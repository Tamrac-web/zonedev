# Executing Plans

## 概述

直接在当前会话中执行小型计划。加载计划，快速审查，逐任务执行，完成后收尾。

**适用场景：** ≤2 个任务的小型计划，由 writing-plans 自动路由到此流程。对于 3 个以上任务的标准计划，使用 zonedev:subagent-driven-development。

## 流程

### 步骤 1：加载并审查计划
1. 读取计划文件
2. 快速审查——找出计划中的任何问题
3. 如有疑虑：在开始前向用户提出
4. 如无疑虑：开始执行

### 步骤 2：逐任务执行

对于每个任务：
1. 严格按照计划步骤执行
2. 遵循 TDD 循环（Red-Green-Refactor）
3. 按规定运行验证
4. 提交

### 步骤 3：完成开发

所有任务完成并验证后：
- **必需的子 skill：** 使用 zonedev:finishing-a-development-branch
- 验证测试、展示选项、执行选择

## 何时停下来

- 遇到阻塞（缺少依赖、测试失败、指令不清）
- 验证反复失败
- 发现计划规模超出预期（改用 zonedev:subagent-driven-development）

**宁可请求澄清，也不要猜测。**

## 要点

- 严格按照计划步骤执行
- 不要跳过验证
- 被阻塞时停下来，不要猜测
- 未经用户明确同意，绝不在 main/master 分支上开始实现

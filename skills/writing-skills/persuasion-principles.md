# Skill 设计中的说服原则

## 概述

LLM 和人类一样受到相同说服原则的影响。理解这些心理学原理有助于你设计更有效的 skill——不是为了操纵，而是为了确保关键实践在压力下也能被遵循。

**研究基础：** Meincke 等人（2025）在 N=28,000 次 AI 对话中测试了 7 种说服原则。说服技术使遵从率提高了一倍以上（33% → 72%，p < .001）。

## 七大原则

### 1. 权威性
**定义：** 对专业知识、资质或官方来源的顺从。

**在 skill 中的运作方式：**
- 命令式语言："你必须"、"永远不要"、"始终"
- 不可协商的框架："没有例外"
- 消除决策疲劳和合理化倾向

**何时使用：**
- 纪律强制型 skill（TDD、验证要求）
- 安全关键的实践
- 已确立的最佳实践

**示例：**
```markdown
✅ 先写代码再写测试？删掉它。重新来过。没有例外。
❌ 在可行时考虑先写测试。
```

### 2. 承诺一致性
**定义：** 与先前行为、声明或公开宣告保持一致。

**在 skill 中的运作方式：**
- 要求宣布："宣布 skill 使用情况"
- 强制明确选择："选择 A、B 或 C"
- 使用跟踪：TodoWrite 用于检查清单

**何时使用：**
- 确保 skill 真正被遵循
- 多步骤流程
- 问责机制

**示例：**
```markdown
✅ 当你找到一个 skill 时，你必须宣布："我正在使用 [Skill 名称]"
❌ 考虑让你的用户知道你正在使用哪个 skill。
```

### 3. 稀缺性
**定义：** 来自时间限制或有限可用性的紧迫感。

**在 skill 中的运作方式：**
- 时间约束的要求："在继续之前"
- 顺序依赖："在 X 之后立即"
- 防止拖延

**何时使用：**
- 即时验证要求
- 时间敏感的工作流
- 防止"我以后再做"

**示例：**
```markdown
✅ 完成任务后，立即请求代码审查，然后再继续。
❌ 你可以在方便的时候审查代码。
```

### 4. 社会认同
**定义：** 遵从他人的做法或被视为正常的行为。

**在 skill 中的运作方式：**
- 普遍模式："每次都是"、"始终"
- 失败模式："没有 Y 的 X = 失败"
- 建立规范

**何时使用：**
- 记录普遍实践
- 警示常见失败
- 强化标准

**示例：**
```markdown
✅ 没有 TodoWrite 跟踪的检查清单 = 步骤会被跳过。每次都是。
❌ 有些人觉得 TodoWrite 对检查清单有帮助。
```

### 5. 团结感
**定义：** 共同身份，"我们是一伙的"，群体归属。

**在 skill 中的运作方式：**
- 协作语言："我们的代码库"、"我们是同事"
- 共同目标："我们都想要质量"

**何时使用：**
- 协作工作流
- 建立团队文化
- 非等级化的实践

**示例：**
```markdown
✅ 我们是一起工作的同事。我需要你诚实的技术判断。
❌ 你大概应该告诉我如果我错了。
```

### 6. 互惠
**定义：** 回报所获利益的义务感。

**运作方式：**
- 谨慎使用——可能让人感觉是在操纵
- 在 skill 中很少需要

**何时避免：**
- 几乎总是（其他原则更有效）

### 7. 喜好
**定义：** 倾向于与我们喜欢的人合作。

**运作方式：**
- **不要用于合规**
- 与诚实反馈文化冲突
- 制造谄媚

**何时避免：**
- 纪律强制时始终避免

## 按 Skill 类型的原则组合

| Skill 类型 | 使用 | 避免 |
|------------|------|------|
| 纪律强制型 | 权威性 + 承诺一致性 + 社会认同 | 喜好、互惠 |
| 指导/技术型 | 适度权威性 + 团结感 | 强权威 |
| 协作型 | 团结感 + 承诺一致性 | 权威性、喜好 |
| 参考型 | 仅需清晰 | 所有说服原则 |

## 为什么有效：心理学解释

**明确的规则减少合理化：**
- "你必须"消除决策疲劳
- 绝对化语言消除"这是不是例外？"的问题
- 明确的反合理化措施封堵特定漏洞

**执行意图创造自动行为：**
- 清晰的触发条件 + 要求的行动 = 自动执行
- "当 X 时，做 Y"比"通常做 Y"更有效
- 降低合规的认知负担

**LLM 是准人类的：**
- 在包含这些模式的人类文本上训练
- 权威性语言在训练数据中先于合规
- 承诺一致性序列（声明 → 行动）被频繁建模
- 社会认同模式（大家都做 X）建立规范

## 伦理使用

**正当的：**
- 确保关键实践被遵循
- 创建有效的文档
- 防止可预见的失败

**不正当的：**
- 为个人利益操纵
- 制造虚假紧迫感
- 基于愧疚的合规

**检验标准：** 如果用户完全理解这个技术，它是否仍然服务于用户的真正利益？

## 研究引用

**Cialdini, R. B. (2021).** *Influence: The Psychology of Persuasion (New and Expanded).* Harper Business.
- 七大说服原则
- 影响力研究的实证基础

**Meincke, L., Shapiro, D., Duckworth, A. L., Mollick, E., Mollick, L., & Cialdini, R. (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.
- 在 N=28,000 次 LLM 对话中测试了 7 种原则
- 说服技术使遵从率从 33% 提高到 72%
- 权威性、承诺一致性、稀缺性最有效
- 验证了 LLM 行为的准人类模型

## 快速参考

设计 skill 时，问自己：

1. **它是什么类型？** （纪律型 vs. 指导型 vs. 参考型）
2. **我试图改变什么行为？**
3. **哪些原则适用？** （纪律型通常使用权威性 + 承诺一致性）
4. **是否组合了太多？** （不要使用全部七种）
5. **这是否合乎伦理？** （服务于用户的真正利益？）
